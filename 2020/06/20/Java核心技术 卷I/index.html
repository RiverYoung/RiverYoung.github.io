第三章 Java的程序设计结构

3.1 第一个Java程序

  1. Java区分大小写;
  2. Java使用骆驼命名法(CamelCase);
  3. Java中的main方法必须是静态的 static;
  4. Java中的注释可以使用 //, 也可以使用/* */，但是后者不能嵌套；

3.2 数据类型

类型 存储需求 取值范围 备注
boolean ture/false 整型值不能与布尔值转化
char 2字节 \u0000\uFFFF（065535） 字符类型，使用需要配合单引号 ‘A’
byte 1字节 -128~127 整型
short 2字节 -32768~32767 整型
int 4字节 -2147483648~2147483647 整型
long 8字节 很长，很长 整型
float 4字节 有效位为6~7位 浮点类型
double 8字节 有效位为15位（双精度数值） 浮点类型

  1. float类型较少使用，其数值需要后缀F或f标明（3.14F），否则默认为double；

  2. Java没有无符号形式的byte，short，int或long类型；

  3. char和byte的区别？

    byte 是整型数据 ，是有符号型的，占1 个字节；大小范围为-128~127 ；

    char 是字符数据类型 ，是无符号型的，占2字节(Unicode码 ）；大小范围 是0~65535 ；

    char是一个16位二进制的Unicode字符，JAVA用char来表示单字符，例如 ‘A’ ；

    强烈建议不要在程序中使用char类型

  4. 数值之间的转换

    1. 无信息丢失的转换，eg: byte → short → int → long/double; char → int; float → double

    2. 有精度损失的转换，eg: int → float; long → double；long → float;

    3. 强制类型转换

3.3 变量与常量

  1. 千万不能使用未初始化的变量的值，会报错；

  2. 变量的声明尽可能的靠近变量第一次使用的地方，这是一种良好的程序编写风格；

  3. 常量用关键字 final 声明，常量名使用全大写，一旦赋值无法修改；

    如果希望某个常量可以被一个类中的多个方法使用，可以用关键字 static final 设置为类常量(class constant)，类常量要定义在main方法的外部，如果声明为public，则可以再其它类的方法使用

  4. 可以将变量的取值只在一个有限的集合内的数值使用枚举类型 enum保存；

  5. 在Math类中，提供了很多方便计算的方法和常量

  6. && 和 ||是按照“短路”方法求值的，比如 A&&B 和 C||D中 A为假或者C为真时，B和D就不会执行；但是&和|不会按照“短路”规则运算；

3.4 字符串

  1. Java没有预置字符串类型，String是标准Java类库中提供的预定义类；

  2. 每个用双引号括起来的字符串都是String类的一个实例；

  3. 不允许修改Java字符串中的单个字符，String类对象是不可变的；

  4. Java中的String类提供了丰富的API接口

  5. 如果需要频繁的拼接字符串，那么使用StringBuilder类可以避免每次拼接字符串的时候都要重新构建一个String对象，既耗时又浪费空间

3.5 数组

  1. 数组是一种数据结构，用来存储同一类型的数值的集合。

  2. 循环处理数组中的每个元素

  3. 数组常见API

  4. 如果程序运行中需要经常扩展数组的大小，则应该使用另一种数据结构 – 数组列表（array list）；

3.6 输入与输出

  1. 控制台输出最常见的就是 System.out.println("Hello world"); 当然还有格式化输出（Java 5）

  2. 控制台输入，需要构造一个与“标准输入流” System.in 关联的Scanner对象

  3. 文件的输入与输出

第四章 对象与类

4.1 面向对象程序设计概述

​ 面向对象程序（object-oriented programming，OOP）是由对象组成的，每个对象都包含对用户公开的特定功能部分和隐藏的实现部分。传统的结构化程序设计是通过一系列的过程（即算法）来解决问题，确定了这些过程后，然后再考虑用适当的方式存储数据，这也是为什么Pascal语言的设计者Niklaus Wirth将其著作命名为《算法+数据结构=程序》的原因（算法第一，数据结构是第二位的）。而OOP调换了这个顺序，将数据放在第一位，然后才考虑操作数据的算法。

​ 类(class) 是构造对象的模板或蓝图。由类构造(construct)对象的过程称为创建类的实例(instance);

​ 封装(encapsulation) 是处理对象的一种重要概念，将数据和行为组合在一个包中，并对对象的使用者隐藏具体的实现方式。对象中的数据称为实例字段(instance field)，操作数据的过程称为方法(method)，作为类的实例，特定对象都有一组特定的实例字段值，这些值得集合就是这个对象的状态(state)。

​ 继承(inhertance)就是通过扩展一个类来新建另一个类的过程，新建类会拥有被扩展类的全部属性及方法。

​ 对象(object)都有三个特征：行为，状态和标识。

​ 类之间的关系：

  • 依赖(“use-a”)：一个类的方法使用另一个类的对象，就认为这个类依赖于另一个类，（平常建议尽可能的较少互相依赖的类，这样子能较少类之间的耦合）；

  • 聚合(“has-a”)：类A的对象包含类B的对象；

  • 继承(“is-a”)：一个特殊的类与一个更一般的类之间的关系；

    对应的UML符号

4.2 构造器

​ 要使用对象，首先要构造对象，并初始化其初始状态，然后对对象应用方法。而构造对象就是由构造器(constructor)完成构造并初始化的。

​ 构造器的注意点：

  1. 构造器的名字应该与类名相同；

  2. 每个类可以有一个以上的构造器；

    ​ 重装(overloading)：多个方法具有相同名称，但是不同的参数；因此要完整的描述一个方法，需要指定方法名及参数类型，这些叫做方法的签名(signature)，返回值不是方法签名的一部分。

    ​ 如果一个类没有编写构造器，那么系统会为你提供一个无参数构造器，并将所有的实例字段设置成默认值(数值为0，布尔值为false，对象引用为null)，但是还是建议不要依赖于默认值的做法，还是需要自己进行初始化，提高程序可读性。

    ​ 实例字段初始化的方法有：

    • 在构造器中设置值；
    • 在类定义中直接为任何字段赋值，该动作会在执行构造器之前完成，如果一个类的所有构造器都希望把某个特定的实例字段设置为一个值，那么这个语法就特别有用；
    • 初始化块(initialization block)：不常见，通常还是将初始化代码放置在构造器中；

  3. 构造器没有返回值；

  4. 构造器总是伴随着new操作符一起调用；

​ ps: 在析构器中，常见的操作是回收分配给对象的存储空间，但由于Java会完成自动的垃圾回收，不需要人工回收内存，所以Java不支持析构器。

更改器方法(mutator method)：调用这个方法后，对象的状态会改变；

访问器方法(accessor method)：只访问对象而不修改对象的方法；

4.3 常用关键字

4.3.1 final

​ 将实例字段定义为 final，这样的字段必须在构造对象时初始化，也就是说确保在每一个构造器执行之后，这个字段的值已经设置，并且以后不会再修改这个字段了。

4.3.2 static

​ 静态字段

​ 将一个字段定义为 static，每个类中只有一个这样的字段，它属于类而不属于任何单个的对象。而对于非静态的实例字段，每个对象都有自己的一个副本。

​ 静态常量

​ 静态变量用的少，但是静态常量却很常用。

​ 静态方法

​ 静态方法是不在对象上执行的方法，可以认为静态方法是没有this参数的方法（在一个非静态的方法中，this参数是这个方法的隐式参数），所以不可以访问非静态方法的实例字段；

​ 虽然可以使用对象调用静态方法，但是这种写法容易造成混淆，因为往往对象调用的静态方法得到的结果跟该对象无关，所以还是建议使用类名而不是对象来调用静态方法；

C 引入关键字static是为了表示退出与一个块后依量，此时术语“静态”是有意义的：变量一直保留，当再次进入该块时仍然存在；static 在C中的另一个含义是表示不能从其它文件访问的全局变量和函数，为了避免引入一个新的关键字就继续沿用了；C++ 第三次沿用了该关键字则和前面的含义无关，它指示属于类而不属于任何类对象的变量和函数，这个含义和Java中的关键字含义一致。
