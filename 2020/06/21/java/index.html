Java核心技术 卷I

第三章 Java的程序设计结构

3.1 第一个Java程序

  1. Java区分大小写;

  2. Java使用骆驼命名法(CamelCase);

  3. Java中的main方法必须是静态的 static;

  4. Java中的注释可以使用 //, 也可以使用/* */，但是后者不能嵌套；

3.2 数据类型

类型 存储需求 取值范围 备注
boolean ture/false 整型值不能与布尔值转化
char 2字节 \u0000\uFFFF（065535） 字符类型，使用需要配合单引号 ‘A’
byte 1字节 -128~127 整型
short 2字节 -32768~32767 整型
int 4字节 -2147483648~2147483647 整型
long 8字节 很长，很长 整型
float 4字节 有效位为6~7位 浮点类型
double 8字节 有效位为15位（双精度数值） 浮点类型

  1. float类型较少使用，其数值需要后缀F或f标明（3.14F），否则默认为double；

  2. Java没有无符号形式的byte，short，int或long类型；

  3. char和byte的区别？

    byte 是整型数据 ，是有符号型的，占1 个字节；大小范围为-128~127 ；

    char 是字符数据类型 ，是无符号型的，占2字节(Unicode码 ）；大小范围 是0~65535 ；

    char是一个16位二进制的Unicode字符，JAVA用char来表示单字符，例如 ‘A’ ；

    强烈建议不要在程序中使用char类型

  4. 数值之间的转换

    1. 无信息丢失的转换，eg: byte → short → int → long/double; char → int; float → double

    2. 有精度损失的转换，eg: int → float; long → double；long → float;

    3. 强制类型转换

3.3 变量与常量

  1. 千万不能使用未初始化的变量的值，会报错；

  2. 变量的声明尽可能的靠近变量第一次使用的地方，这是一种良好的程序编写风格；

  3. 常量用关键字 final 声明，常量名使用全大写，一旦赋值无法修改；

    如果希望某个常量可以被一个类中的多个方法使用，可以用关键字 static final 设置为类常量(class constant)，类常量要定义在main方法的外部，如果声明为public，则可以再其它类的方法使用

  4. 可以将变量的取值只在一个有限的集合内的数值使用枚举类型 enum保存；

  5. 在Math类中，提供了很多方便计算的方法和常量

  6. && 和 ||是按照“短路”方法求值的，比如 A&&B 和 C||D中 A为假或者C为真时，B和D就不会执行；但是&和|不会按照“短路”规则运算；

3.4 字符串

  1. Java没有预置字符串类型，String是标准Java类库中提供的预定义类；

  2. 每个用双引号括起来的字符串都是String类的一个实例；

  3. 不允许修改Java字符串中的单个字符，String类对象是不可变的；

  4. Java中的String类提供了丰富的API接口

  5. 如果需要频繁的拼接字符串，那么使用StringBuilder类可以避免每次拼接字符串的时候都要重新构建一个String对象，既耗时又浪费空间

3.5 数组

  1. 数组是一种数据结构，用来存储同一类型的数值的集合。

  2. 循环处理数组中的每个元素

  3. 数组常见API

  4. 如果程序运行中需要经常扩展数组的大小，则应该使用另一种数据结构 – 数组列表（array list）；

3.6 输入与输出

  1. 控制台输出最常见的就是 System.out.println("Hello world"); 当然还有格式化输出（Java 5）

  2. 控制台输入，需要构造一个与“标准输入流” System.in 关联的Scanner对象

  3. 文件的输入与输出

第五章 继承

5.1 概念

​ 继承（inheritance）：复用已有的类的属性和方法，并新增自己的属性和方法的技术。已有的类成为父类（parent class），新类称为子类（child class）。用关键字 extends 表示继承。

java用关键字extends替代了C++中的冒号 ( : )，在java中，所有的继承都是公共继承，而没有C++中的私有继承和保护继承

5.2 关键字 final

  1. final常量，常量名使用全大写，一旦赋值无法修改；

  2. final 方法，类中某个方法声明为final方法后，其子类不能覆盖该方法;

  3. final 类，不允许扩展的类。将类声明为final类之后，所有的方法自动称为final 方法，但是不包括属性。

    强制类型转换

    进行强制类型转换的原因：

    ​ 要暂时忽视对象的实际类型之后，使用对象的全部功能。比如，为了让对象引用能是使用子类的特殊处理，需要将对象引用从引用父类通过强制类型转化为子类。

    需要注意点：

    ​ 强制类型转换只适用于继承层次内;

    ​ 将子类的引用赋给父类变量，编译器是允许的；但将一个父类的引用赋给一个子类的变量时，就承诺过多了，必须进行强制类型转换;

    ​ 在进行强制类型转换之前，可以通过使用instanceof操作符检查是否能成功转换;

第六章 接口、Lambda表达式与内部类

接口 interface，用来描述类应该做什么，而不指定它具体怎么做；

一个类可以实现 implements 一个或多个接口；

接口中的所有方法都自动是public方法，因此，在接口中声明方法时，不需提供关键字public；

接口中还可以定义常量，并且会自动默认为public static final；

但是没有实例字段和实例方法，可以将接口看成是没有实例字段的抽象类；

让类实现一个接口时，除了将类声明为实现给定的接口外，还需对接口中的所有方法提供定义；

接口与抽象类

已经有抽象类了，为什么还需要接口？

因为使用抽象类表示通用属性存在一个严重的问题，就是每个类只能扩展一个类。假设某个类扩展了该抽象类，就无法扩展第二个类了。但是，每个类可以实现多个接口。C++中允许一个类多重继承，而Java中选择不支持多重继承，主要原因是多重继承会让语言变得非常复杂或者效率低下。实际上，接口可以提供多重继承的好处，同时避免多重继承的复杂性和低效性。
